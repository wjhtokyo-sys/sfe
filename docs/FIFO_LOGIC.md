# FIFO 业务逻辑说明（当前实现）

本文档描述 SFE 项目中“进货单盘点完成后”的 FIFO 分配与挂起规则。

## 1. 触发点

在超级管理员「进货单管理」页面，点击某进货单的【盘点完成】按钮：

- 接口：`POST /api/purchase-orders/{po_id}/status`
- 入参：`{ status: "checked_inbound" }`

## 2. 处理对象

按该进货单下的每一条进货明细（`purchase_order_lines`）逐行处理，核心字段：

- `JAN`
- `品名`
- `数量`
- `进货单价`

## 3. 货品与订单的区分

- **货品（到货）**：来自进货单明细（`purchase_order_lines`）
- **订单（需求）**：来自客户订单（`customer_orders`）

FIFO 的本质是：将“到货货品”按规则匹配到“客户订单需求”。

## 4. FIFO 判定规则（当前）

对每条进货明细先按 JAN 找到商品（`items`），再找该商品下未完成订单（`status=open`）。

### 4.1 多客户命中（优先拦截）

若同一 JAN 命中 **2 个及以上客户** 的未完成订单：

- 立即取消自动分配
- 不自动入库分配到订单
- 生成挂起任务到 `fifo_pending_tasks`
  - `reason_code = multi_customer_match`
  - `reason_text = JAN命中多个客户未完成订单，需人工干预`

### 4.2 无匹配

若同一 JAN 命中 **0 个**未完成订单：

- 不自动分配
- 生成挂起任务到 `fifo_pending_tasks`
  - `reason_code = no_order_match`
  - `reason_text = 未命中客户订单，需人工处理`

### 4.3 单客户命中且“新到货数量 <= 订单剩余数量”

若同一 JAN 仅命中 **1 个客户**，且本次新到货数量小于等于该客户订单剩余数量：

- 自动入库生成库存批次（`inventory_lots`）
- 自动按订单创建时间升序进行分配（FIFO）
- 生成分配记录（`allocations`）
- 更新订单已分配数量与状态（`open/closed`）

### 4.4 单客户命中且“新到货数量 > 订单剩余数量”

若同一 JAN 仅命中 **1 个客户**，但本次新到货数量大于订单剩余数量：

- 先按 FIFO 对该订单做可分配部分
- 该订单满足后，超出的到货数量不直接入常规自动分配
- 超出部分进入 FIFO 管理页的「无匹配货品管理」
- 由人工在 FIFO 页面继续处理

## 5. FIFO 管理页结构

页面：超级管理员 → FIFO管理

分为两块：

1. **多客户命中分配管理**
2. **无匹配货品管理**

两块表格统一展示列：

- 进货单号
- 进货日期
- JAN
- 品名
- 数量
- 进货价格
- 操作

## 6. 防重复策略

生成挂起任务时，系统会检查同一进货明细在同一原因下是否已有 `pending` 任务：

- 若已存在，不重复插入
- 避免重复盘点导致挂起任务无限叠加

## 7. 相关数据表

- `purchase_orders`
- `purchase_order_lines`
- `items`
- `customer_orders`
- `inventory_lots`
- `allocations`
- `fifo_pending_tasks`

## 8. 多客户命中分配管理（界面与执行规则）

适用页面：超级管理员 → FIFO管理 → 多客户命中分配管理

### 8.1 操作列组件

原“执行动作下拉”调整为以下组件（从左到右）：

1. **客户名与订单信息下拉菜单（必填）**
   - 仅展示该 JAN 可匹配的未完成订单
   - 展示格式：`<客户名> + <下单日期> 下单 <数量>点`
   - 推荐内部值：`order_id`

2. **数量输入框（必填）**
   - 本次要匹配的数量（正整数）

3. **匹配按钮**
   - 按钮名称：`匹配`（原“执行”）

### 8.2 点击“匹配”前校验

以下条件需同时满足：

- 已选择“客户名与订单信息下拉菜单”
- 已填写“数量”
- 数量 > 0 且为整数
- 数量 <= 当前挂起行剩余数量
- 数量 <= 目标订单剩余需求（`qty_requested - qty_allocated`）

任一不满足：禁止执行并提示错误。

### 8.3 匹配执行规则

点击“匹配”后：

1. 按 `order_id + qty` 生成分配记录（Allocation）
2. 更新目标订单：
   - `qty_allocated += qty`
   - 若达到需求则 `status=closed`，否则保持 `open`
3. 更新当前挂起记录剩余数量：`pending.qty -= qty`
4. 若 `pending.qty > 0`：该行继续显示在“多客户命中分配管理”
5. 若 `pending.qty == 0`：该挂起记录标记为已处理（或从本表消失）

### 8.4 剩余量处理

- 支持同一挂起记录多次匹配（分批处理）
- 每次匹配后立即刷新当前行剩余数量
- 有剩余则保留，无剩余则结束
